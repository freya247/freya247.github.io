Module 2 Artefacts

In the Network and Information Security Management (NISM) module, various cyber security and wider IT concepts were covered across 12 one weeklong units. Each unit is comprised of a combination of reading, discussions, practical activities, seminar presentations and summative reports. All individual and team activities were completed by the set deadlines, including all recommended reading, and I attended every seminar to make the most of my learning journey.

Units 1-3

The first quarter of the module established a foundational knowledge of the history, principles and real world implications of network security. Module teams were formed which led to the creation of a basic cloud-hosted website and the subsequent initial site scanning.

Unit 1 Team Task: Team 3 Contract

Unit 1-3 Individual Task: Collaborative Learning Discussion 1

Unit 2 Team Task: Seminar 1 Presentation - STRIDE and DREAD Tools

Unit 3 Individual Task: Basic Website Scan



Units 4-6

The second quarter of the module took a deeper dive into network architecture and associated security testing best practices. This culminated in not only a testing tools evaluation but also a penetration test design document.

Team Assignment: PDF for test



Units 7-9

The third quarter of the module took a wider lens on topics interlinked with security testing, including impacts on risk, compliance, and the importance of incident response. In addition, security testing skills continued to be practiced and developed through various assignments.

Individual Task: Excel file for show



Units 10-12

The final quarter of the module focussed clearly on data breaches, future internet architectures and security strategy. Both a debate and penetration testing report were successfully completed as a team.

Team Assignment: A GitHub repository/site



The tasks and assignments in this Module have influenced my learning journey in many different ways.
